# Component: `CryptoTable`

## 1. 개요

주요 암호화폐의 실시간 시세 정보를 표 형태로 보여주는 컴포넌트입니다.

## 2. 기능

-   `/api/tickers` API를 호출하여 여러 암호화폐의 현재가, 등락률 등의 정보를 받아옵니다.
-   컴포넌트가 마운트될 때 데이터를 즉시 로드하고, 이후 30초마다 자동으로 데이터를 갱신합니다.
-   데이터 로딩 중에는 "로딩 중..." 메시지를, API 호출 실패 시에는 에러 메시지를 표시합니다.
-   각 암호화폐에 대해 자산명(한글), 심볼, 현재 가격(원화), 24시간 변동률을 표시합니다.
-   변동률은 상승 시 초록색, 하락 시 빨간색으로 표시됩니다.
-   각 행의 '거래' 버튼을 클릭하면 `handleOpenModal` prop으로 전달된 함수를 호출하여 거래 모달을 엽니다.

## 3. 주요 상태(State) 및 Props

### Props

| Prop              | 타입                                       | 필수 | 설명                                                              |
| ----------------- | ------------------------------------------ | ---- | ----------------------------------------------------------------- |
| `handleOpenModal` | `(ticker: Ticker, type: 'buy' | 'sell') => void` | 예   | '거래' 버튼 클릭 시 호출될 함수. 해당 암호화폐 정보와 거래 종류를 인자로 받습니다. |

### 내부 상태

-   `tickers`: API로부터 받아온 실시간 시세 데이터 배열 (`Ticker[]`)
-   `loading`: 데이터 로딩 상태를 관리하는 boolean 값
-   `error`: API 요청 실패 시 에러 메시지를 저장하는 string 값

## 4. 데이터 흐름

1.  컴포넌트가 렌더링되면 `useEffect` 훅에서 `fetchTickers` 함수를 호출하여 시세 정보를 요청하고 `loading` 상태를 `true`로 설정합니다.
2.  `/api/tickers`로부터 데이터를 성공적으로 받아오면 `tickers` 상태를 업데이트하고 `loading`을 `false`로, `error`를 `null`로 설정합니다.
3.  API 요청이 실패하면 `error` 상태에 에러 메시지를 저장합니다.
4.  `setInterval`을 사용하여 30초마다 `fetchTickers` 함수를 반복 호출하여 데이터를 최신 상태로 유지합니다.
5.  컴포넌트가 언마운트될 때 `clearInterval`을 호출하여 인터벌을 정리합니다.
6.  사용자가 '거래' 버튼을 클릭하면, 해당 행의 `ticker` 정보와 거래 타입('buy')을 인자로 하여 `handleOpenModal` 함수를 실행합니다.
